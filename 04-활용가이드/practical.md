# 개요

> **앞서 소개한 도구들을 바탕으로 실제 결과물을 생성하는 방법을 안내합니다.** 만들고 싶은 결과물이 있을 때, 해당 카테고리에 들어가면 어떤 도구를 어떻게 쓰면 되는지 확인할 수 있습니다.
>
> 제너레이팅 카테고리는 다음과 같습니다.
> 1. PPT 제작
> 2. 카드뉴스 제작
> 3. 카드뉴스 자동화
> 4. 포스터 / 배너
> 5. 로고 디자인
> 6. 랜딩페이지 제작
> 7. 와이어프레임



# PPT 제작

> 주제를 빠르게 시각 자료로 바꿔야 할 때, AI PPT 도구로 초안을 먼저 만들고 사람이 최종 완성도를 올리는 방식이 가장 효율적입니다.

- 추천 도구
  1. [Gamma](https://gamma.app/) — 프롬프트 기반으로 PPT 초안을 빠르게 생성합니다.
  2. [Snapdeck](https://snapdeck.app) — 문서/텍스트를 슬라이드 구조로 변환할 때 유용합니다.
  3. Canva — 템플릿 기반으로 디자인 톤을 빠르게 맞춥니다.
  4. Genspark — 리서치+생성+정리까지 한 번에 진행하기 좋습니다.
- 기본 워크플로우
  1. 주제와 청중(발표 대상/목적)을 먼저 정합니다.
  2. AI 도구로 목차와 초안 슬라이드를 생성합니다.
  3. 핵심 메시지, 수치, 사례를 직접 검수·보완합니다.
  4. 템플릿/폰트/컬러를 통일해 가독성을 맞춥니다.
  5. PPT/PDF로 내보내고 최종 리허설합니다.

# 카드뉴스 제작

> 피그마 템플릿을 활용하면 카드뉴스를 처음 만드는 분도 빠르게 결과물을 완성할 수 있습니다.

- 기본 흐름(피그마)
  1. 템플릿을 불러옵니다.
  2. 배경색을 콘텐츠 톤에 맞게 변경합니다.
  3. 제목/본문 폰트를 브랜드 톤에 맞게 교체합니다.
  4. 이미지를 넣고 구도를 맞춥니다.
  5. 최종 저장(Export)합니다.
- 자주 쓰는 단축키
  - `Ctrl + Shift + K` : 이미지 넣기
  - `Alt + 더블클릭` : 이미지 위치/크롭 편집
  - `Ctrl + Shift + C` : 프레임을 PNG로 바로 복사
- 참고: 자세한 단계별 실습은 **02-꿀팁모음(tips.md) 인스타그램 템플릿 아티클**을 확인하면 됩니다.
- 대체 도구: Canva, 미리캔버스로도 동일한 흐름(템플릿 수정 → 이미지/텍스트 교체 → 저장)으로 제작할 수 있습니다.

# 포스터 / 배너

> 포스터·배너는 목적에 맞는 유형을 먼저 고르면 제작 속도와 완성도가 함께 올라갑니다.

- 유형별 제작 가이드
  1. **텍스트 강조형**: 큼직한 텍스트 + 강렬한 컬러로 이목을 끕니다.
     - 행사 공지, 모집, 마감 임박 안내에 적합합니다.
  2. **이미지 강조형**: 상품/콘텐츠 이미지를 크게 배치합니다.
     - 제품 소개, 공간 소개, 포트폴리오 홍보에 적합합니다.
  3. **3D/누끼 활용형**: 3D 에셋, 이모지, 누끼 요소로 시선을 만듭니다.
     - 이벤트/프로모션, 트렌디한 SNS 배너에 적합합니다.
- 이런 상황에서 이렇게 쓰면 좋습니다.
  - 빠른 시안 다량 제작: Canva, 미리캔버스
  - 세밀한 편집/컴포넌트 관리: 피그마
  - 텍스트 중심 비주얼 생성: Ideogram
- 실무 팁: 먼저 레퍼런스로 톤을 정한 뒤(텍스트형/이미지형/3D형), 한 유형으로 2~3개 시안을 짧게 비교 제작합니다.

# 로고 디자인

> 로고를 빠르게 탐색하고 시안을 뽑아보는 작업 흐름을 정리한 섹션입니다.

로고 디자인은 무작정 그리기보다, 먼저 방향을 정하는 것이 중요합니다. 세리프 로고로 갈지, 산세리프 로고로 갈지, 심볼 중심으로 갈지, 워드마크 중심으로 갈지를 먼저 정하셔야 합니다. 방향을 잡은 뒤에는 Behance, Pinterest, Fonts In Use 같은 레퍼런스 사이트에서 분위기를 충분히 수집하고, 이후 아래 도구들로 빠르게 시안을 생성해볼 수 있습니다.

1. [Ideogram](https://ideogram.ai/t/explore?f=logo)
2. [Brandmark](https://chat.brandmark.io/templates)
3. [LogoAI](https://www.logoai.com/logo/)
4. [logo.com](https://logo.com/)
5. [Genspark](https://www.genspark.ai)
6. [Gemini 나노바나나](https://gemini.google.com/)

핵심은 **레퍼런스 → 방향 설정 → AI 시안 생성 → 피드백 → 정리** 순서로 가는 것입니다. 처음부터 완성형 로고를 만들려고 하기보다, 다양한 시안을 빠르게 본 뒤 팀 안에서 가장 SPEC에 맞는 방향을 고르는 방식이 훨씬 효율적입니다. 특히 logo.com은 빠른 워드마크 탐색에, Genspark와 Gemini 나노바나나는 다양한 스타일 시안 탐색에 활용하기 좋습니다.

# 랜딩페이지 제작

> 랜딩페이지는 "구조를 빠르게 만들고, 핵심 메시지를 반복 검수"하는 방식으로 접근하면 시행착오를 줄일 수 있습니다.

- 추천 방식 1: **Pencil Dev + Claude Code**
  - 프롬프트 기반으로 섹션 구조(히어로/문제/해결/후기/CTA)를 빠르게 생성합니다.
  - 카피와 레이아웃을 짧은 주기로 수정하며 완성도를 올립니다.
- 추천 방식 2: **Figma → 코드 변환 워크플로우**
  1. Figma에서 와이어/시안을 먼저 정리합니다.
  2. 핵심 컴포넌트(버튼, 카드, 섹션)를 일관되게 구성합니다.
  3. 코드 변환 도구 또는 AI 코딩으로 구현하고 반응형을 보정합니다.
- 참고: 빠른 검증이 목표라면 Webflow, Framer 같은 노코드 도구로 MVP 랜딩을 먼저 공개하는 것도 좋은 선택입니다.

# 와이어프레임

> 와이어프레임은 "디자인"보다 먼저 "구조와 흐름"을 정하는 단계이며, 초반 합의 비용을 크게 줄여줍니다.

- 기본 도구: **Figma**
- 초보자 시작 순서
  1. 화면 목적과 사용자 행동(클릭/입력/전환)을 먼저 적습니다.
  2. 저해상도 박스 중심으로 정보 구조를 배치합니다.
  3. 화면 간 이동 흐름을 연결해 빠르게 리뷰합니다.
- 생산성 높이는 기능(팁 아티클 PM 파트 연계)
  - **오토 레이아웃**: 텍스트 길이가 바뀌어도 레이아웃이 자동 정렬됩니다.
  - **컴포넌트**: 반복 UI를 통일해 수정 시간을 줄입니다.
- 실무 팁: 와이어 단계에서는 색/디테일보다는 정보 우선순위와 CTA 위치 검증에 집중합니다.
