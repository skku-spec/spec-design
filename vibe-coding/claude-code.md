# Claude Code
<!-- new -->

> **Claude Code는 Anthropic이 만든 코딩 에이전트입니다.** 터미널에서 한국어로 "이런 거 만들어줘"라고 말하면, AI가 실제로 작동하는 코드를 만들어줍니다. 코딩 경험이 전혀 없어도 괜찮습니다.
>
> 이 챕터에서 배우실 내용은 다음과 같습니다.
> 1. Claude Code란 무엇인가
> 2. 설치 & 세팅 (Node.js, CLI)
> 3. 첫 실행과 기본 조작법
> 4. 결과 확인하기 (npm, localhost)
> 5. 수정과 이어서 작업하기
> 6. 명령어 & 단축키 모음
> 7. 트러블슈팅

## Claude Code란?

Claude Code는 터미널(명령어 입력 창)에서 동작하는 AI 도구입니다. 일반 채팅 AI와 달리, **파일을 직접 만들고, 수정하고, 실행까지** 합니다.

### 바이브코딩과의 관계

바이브코딩은 AI에게 자연어로 지시해서 코드를 만드는 방식입니다. Claude Code는 그 바이브코딩을 실행하는 **도구**입니다.

| 구분 | 전통적인 코딩 | 바이브코딩 (Claude Code) |
|------|-----------|----------------------|
| 방식 | 개발자가 코드를 직접 작성 | 한국어로 설명하면 AI가 코드 작성 |
| 필요 역량 | 프로그래밍 언어 숙달 | 원하는 것을 명확히 설명하는 능력 |
| 역할 | 개발자 = 작업자 | 나 = 감독, AI = 작업자 |

### 할 수 있는 것

1. **파일 생성/수정** — "index.html 만들어줘"라고 하면 실제 파일을 만듭니다.
2. **프로젝트 전체 이해** — 폴더 안의 모든 파일을 읽고, 구조를 파악합니다.
3. **프로그램 실행** — 만든 코드를 바로 실행해서 결과를 확인합니다.
4. **오류 수정** — 에러가 나면 스스로 원인을 파악하고 고칩니다.
5. **연속 작업** — "이거 만들고, 저거도 수정하고, 테스트까지 해줘"처럼 여러 단계를 한 번에 처리합니다.

### 만들 수 있는 것과 없는 것

**만들 수 있는 것 (추천)**

- 웹사이트/웹페이지 — 소개 페이지, 포트폴리오, 랜딩페이지, 이벤트 안내
- 업무 자동화 도구 — 계산기, 일정 캘린더, 예약 시스템, 설문 폼
- 간단한 프로젝트 — 명언 생성기, D-day 계산기, 날씨 앱, 가위바위보 게임

**도전적인 것 (가능하지만 시간 필요)**

- 결제 시스템, 회원가입/로그인, 대용량 파일 처리, 실시간 채팅

**전문가가 필요한 것 (비추천)**

- 은행 수준 보안, 수천 명 동시 접속 서비스, 법적 책임이 따르는 시스템

## 설치 & 세팅

### 준비물 체크리스트

| 필요한 것 | 설명 |
|---------|------|
| **컴퓨터** | Mac 또는 Windows 둘 다 가능 |
| **인터넷** | WiFi 연결 |
| **Claude 계정** | [anthropic.com](https://anthropic.com)에서 가입, 최소 Pro 플랜 결제 필요 |
| **시간** | 처음엔 2~3시간 정도 여유 |

영어 실력이나 개발 경험은 **필요 없습니다.**

### 1단계: 터미널 열기

터미널은 **컴퓨터한테 글로 명령하는 창**입니다. 마우스 대신 글자로 지시하는 방식이라 처음엔 낯설 수 있지만, Claude Code 안에서는 한국어로 말해도 알아서 처리해줍니다.

**Mac에서 여는 법**

1. `Command + Space` 키를 누릅니다.
2. "터미널" 또는 "Terminal"을 입력합니다.
3. `Enter` 키를 누릅니다.

**Windows에서 여는 법**

1. `Windows 키`를 누릅니다.
2. "PowerShell"을 입력합니다.
3. `Enter` 키를 누릅니다.

### 2단계: Node.js 설치

Node.js는 Claude Code가 작동하는 데 필요한 기본 프로그램입니다. 무료입니다.

1. [nodejs.org](https://nodejs.org) 접속
2. 큰 초록색 버튼 클릭 (LTS 버전)
3. 다운로드된 파일 실행
4. "계속" 또는 "Next" 버튼만 누르면 설치 완료

**설치 확인:** 터미널에서 아래 명령어를 입력하세요.

`node --version`

버전 숫자(예: v24.11.1)가 나오면 정상입니다.

### 3단계: Claude CLI 설치

터미널에서 아래 명령어를 복사해 붙여넣기하고 `Enter`를 누르세요.

`npm install -g @anthropic-ai/claude-cli`

1~2분 기다리면 됩니다. 뭔가 주르륵 지나가는데, 에러 없이 끝나면 성공입니다.

**설치 확인:** 터미널에서 아래 명령어를 입력하세요.

`claude --version`

버전 번호가 나오면 설치 완료입니다.

**"command not found" 에러가 뜨나요?** 터미널을 완전히 닫고 다시 열어보세요. 그래도 안 되면 컴퓨터를 재시작한 뒤 다시 시도하세요.

### 요금제 안내

| 플랜 | 가격 | 사용량 | 추천 대상 |
|------|------|--------|---------|
| **Claude Pro** | 월 $20 | 주당 40~80시간 | 처음 시작하는 분 (추천) |
| **Claude Max** | 월 $100~200 | 주당 140~480시간 | 대규모 프로젝트, 팀 협업 |

**처음엔 Pro 플랜으로 시작하세요.** 기본 사용법을 익히고, 간단한 프로젝트 2~3개 만들어보고, 내가 계속 쓸지 판단한 뒤에 Max로 업그레이드해도 늦지 않습니다.

- Pro/Max 모두 **월 정액제**입니다. 가입 즉시 결제됩니다.
- 무료 체험은 없습니다.
- 다음 달에 안 쓸 것 같으면 **미리 구독 취소**해두세요. 취소해도 결제일까지 사용 가능합니다.

## 첫 실행과 기본 조작법

### 폴더와 경로

컴퓨터에서 파일은 항상 어딘가에 저장됩니다. 그 "어딘가"를 정확하게 표현한 게 **경로**입니다.

| 실제 세계 | 컴퓨터 |
|---------|------|
| 서울시 강남구 테헤란로 123번지 | /Users/username/Documents/my-project |
| 아파트 단지 안의 각 동 | 폴더 (my-project) |
| 각 집 | 파일 (index.html, style.css) |

**폴더 만들고 이동하기**

| 명령어 | 의미 | 자연어로 말하면 |
|--------|------|------------|
| `mkdir my-first-project` | 새 폴더 만들기 | "my-first-project 폴더 만들어줘" |
| `cd my-first-project` | 폴더로 이동 | "my-first-project로 이동해줘" |
| `pwd` | 현재 위치 확인 | "지금 어디야?" |

직접 Finder(Mac)나 탐색기(Windows)에서 폴더를 만들어도 됩니다.

### Claude Code 시작하기

1. 터미널을 엽니다.
2. 프로젝트 폴더로 이동합니다. (`cd 폴더이름`)
3. 아래 명령어를 입력합니다.

`claude`

### 처음 실행 시 인증

처음 `claude`를 입력하면 **브라우저가 자동으로 열립니다.**

1. Anthropic 로그인 페이지에서 로그인합니다.
2. "허용(Allow)" 버튼을 누릅니다.
3. "인증 완료" 메시지가 나오면 브라우저를 닫고 터미널로 돌아옵니다.

인증이 완료되면 터미널에 이런 화면이 나옵니다.

```
Claude Code v2.~.~
안녕하세요! 무엇을 만들어드릴까요?
> _
```

이 `>` 옆에 원하는 것을 한국어로 입력하면 됩니다.

**인증이 안 되나요?** 브라우저가 안 열리면 터미널에 나온 URL을 직접 복사해서 브라우저에 붙여넣으세요. 계속 실패하면 `claude logout` 입력 후 다시 `claude`를 시도하세요.

### 기본 조작법

| 상황 | 조작 |
|------|------|
| "그래, 해줘" | `Y` 또는 `Enter` |
| "아니, 하지 마" | `n` |
| "지금 멈춰" | `Ctrl + C` |
| "끝낼게" | `/exit` 또는 `Ctrl + D` |
| "지우고 다시 시작" | `/clear` |

Claude가 파일을 만들거나 수정할 때 "파일을 생성할까요? (Y/n)" 같은 메시지가 나옵니다. **대부분의 경우 그냥 Enter만 누르면 됩니다.** Claude가 제안하는 대로 진행하는 게 대부분 맞습니다.

## 첫 대화 시작하기

### 어떻게 말하면 되나요?

친구한테 부탁하듯이 말하면 됩니다.

**좋은 예시**

- "간단한 자기소개 페이지 만들어줘"
- "계산기 만들어줘. 더하기 빼기 곱하기 나누기 다 되게"
- "D-day 계산하는 페이지 만들어줘"

**나쁜 예시**

- "React로 SPA를 구현해줘" — 너무 전문적
- "만들어줘" — 뭘 만들지 안 말함
- "홈페이지" — 너무 막연함

***구체적으로 말할수록 결과가 좋습니다.*** "예쁜 페이지 만들어줘"보다 "하늘색 배경에 내 사진이 가운데 있고, 아래에 자기소개 글이 있는 페이지 만들어줘"가 훨씬 좋은 결과를 냅니다.

### AI가 코드 짜는 과정

요청을 하면 Claude가 코드를 짜기 시작합니다. 화면에 뭔가 주르륵 지나가는데 **당황하지 마세요.**

1. Claude가 코드를 짜는 것을 **기다립니다.** (아무것도 누르지 마세요)
2. 파일 생성 질문에 `Enter` 또는 `Y`를 누릅니다.
3. 여러 파일을 만들면 하나씩 물어보는데, 계속 `Enter`만 누르면 됩니다.
4. "완료" 메시지가 나오면 결과를 확인합니다.

만들어진 파일은 **현재 폴더**에 저장됩니다. Finder(Mac)이나 탐색기(Windows)에서 프로젝트 폴더를 열면 확인할 수 있습니다.

## 결과 확인하기

### npm이란?

npm은 **프로그램을 실행하거나 필요한 도구를 설치하는 명령어**입니다. Claude가 코드 파일을 만들어주면, 이 파일을 실제로 화면에서 보려면 npm 명령어가 필요합니다.

| 비유 | 의미 |
|------|------|
| Claude가 만든 코드 | 요리 재료 |
| `npm install` | 요리에 필요한 도구 준비 |
| `npm run dev` | 요리해서 접시에 담기 |
| 브라우저 | 요리를 먹는 식탁 |

### 자주 쓰는 npm 명령어 2개

**1. npm install** — 필요한 도구 설치

`npm install`

처음 프로젝트를 시작할 때, "module not found" 에러가 날 때, 새 기능을 추가했을 때 사용합니다. Claude한테 "필요한 패키지 설치해줘"라고 말해도 됩니다.

**2. npm run dev** — 미리보기

`npm run dev`

Claude한테 **"로컬 서버 켜줘"**, **"미리보기 보여줘"**, **"실행해줘"**라고 말해도 됩니다.

### localhost란?

서버가 켜지면 이런 주소가 나옵니다.

```
VITE v5.0.0 준비 완료
➜ Local: http://localhost:5173/
```

**localhost는 "내 컴퓨터"를 의미합니다.** 이 주소는 내 컴퓨터에서만 보이고, 다른 사람한테 보내도 보이지 않습니다. 나중에 배포하면 다른 사람도 볼 수 있게 됩니다.

포트 번호(5173, 3000, 8080 등)는 매번 다를 수 있습니다. **터미널에 나온 주소를 그대로 사용하세요.**

### 확인하는 순서

1. Claude한테 "로컬 서버 켜줘" 요청 (또는 직접 `npm run dev` 입력)
2. 터미널에 나온 주소 확인
3. 브라우저가 자동으로 열림 (안 열리면 주소 직접 입력)
4. 내가 만든 페이지 확인

## 수정과 이어서 작업하기

### 수정하기

열려 있는 Claude Code에 수정 사항을 자연어로 말하면 됩니다.

- "배경색 파란색으로 바꿔줘"
- "글자 크기 더 크게 해줘"
- "버튼 추가해줘"
- "이 부분 삭제해줘"

코드를 수정하면 브라우저가 **자동으로 새로고침**됩니다.

### 이미 만든 프로젝트 다시 열기

어제 만들던 프로젝트를 오늘 다시 열어서 수정하고 싶을 때는 이렇게 합니다.

1. **터미널 열기**
2. **프로젝트 폴더로 이동** — `cd Desktop/my-project` (저장한 위치에 따라 다름)
3. **Claude Code 시작** — `claude`
4. **수정 요청** — "배경색 바꿔줘", "버튼 키워줘" 등
5. **결과 확인** — "로컬 서버 켜줘"

폴더 위치가 기억나지 않으면, Finder(Mac)이나 탐색기(Windows)에서 프로젝트 폴더를 찾은 뒤 **터미널 창으로 끌어다 놓으면** 경로가 자동으로 입력됩니다.

***핵심: 한 번 만든 프로젝트는 폴더에 계속 남아있습니다.*** Claude Code를 껐다가 다시 켜도, 폴더만 제대로 찾아가면 언제든 이어서 작업할 수 있습니다.

### Claude가 말을 안 들을 때

1. **더 구체적으로 말하기** — "예쁘게 만들어줘" 대신 "배경은 파란색, 글자는 흰색, 버튼은 둥글게 만들어줘"
2. **예시 보여주기** — "이런 사이트처럼 만들어줘: [URL]" 또는 캡처 이미지 첨부
3. **단계별로 나누기** — 한 번에 다 말하지 말고, "먼저 배경색만 바꿔줘" → "이제 버튼 추가해줘" 순서로
4. **새로 시작하기** — "처음부터 다시 만들어줘. [요구사항]"

## 명령어 & 단축키 모음

### 터미널 기본 명령어

| 명령어 | 의미 | 자연어로 말하면 |
|--------|------|------------|
| `cd 폴더이름` | 폴더 이동 | "폴더이름으로 이동해줘" |
| `cd ..` | 한 단계 위로 | "상위 폴더로 가줘" |
| `pwd` | 현재 위치 확인 | "지금 어디야?" |
| `ls` (Mac) / `dir` (Windows) | 폴더 내용 보기 | "파일 목록 보여줘" |
| `mkdir 폴더이름` | 새 폴더 만들기 | "새 폴더 만들어줘" |

### Claude Code 명령어

| 명령어 | 의미 | 자연어로 말하면 |
|--------|------|------------|
| `claude` | Claude Code 시작 | (직접 입력 필요) |
| `npm install` | 패키지 설치 | "필요한 패키지 설치해줘" |
| `npm run dev` | 미리보기 실행 | "로컬 서버 켜줘" |
| `npm run build` | 배포용 빌드 | "배포용으로 빌드해줘" |

### 유용한 단축키

| 키 | 기능 |
|-----|------|
| `Ctrl + C` | 실행 중인 작업 멈추기 |
| `위/아래 화살표` | 이전 명령어 다시 불러오기 |
| `Tab` | 폴더/파일 이름 자동완성 |
| `Ctrl + R` / `Cmd + R` | 브라우저 새로고침 |

### 퍼미션 댄저러스 모드

퍼미션 댄저러스 모드는 AI가 **매번 확인을 받지 않고 자율적으로 작업**하는 모드입니다.

| 항목 | 일반 모드 | 댄저러스 모드 |
|------|----------|--------------|
| 파일 수정 전 | "수정해도 될까요?" 물어봄 | 바로 수정함 |
| 명령어 실행 전 | "실행해도 될까요?" 물어봄 | 바로 실행함 |
| 장점 | 안전함 | 빠름, 편리함 |
| 단점 | 매번 승인해야 해서 느림 | 작업물이 삭제될 수 있음 |

- 최악의 경우 **작업물이 삭제**될 수 있습니다. 하지만 컴퓨터 자체에 큰 타격이 가는 일은 거의 없습니다.
- 중요한 작업 전에는 **git commit**(저장 포인트 만들기)을 해두면 안전합니다.

## 트러블슈팅

### localhost 안 열릴 때

`npm run dev` 했는데 화면에 아무것도 안 뜰 때:

1. 터미널에 에러 메시지가 있는지 확인합니다.
2. 다른 프로그램이 같은 포트를 쓰고 있을 수 있습니다. `npm run dev -- --port 3001`로 다른 포트를 시도합니다.
3. 터미널을 완전히 끄고 다시 시작합니다.

### 에러 메시지 떴을 때

빨간 글씨가 주르륵 나와도 **당황하지 마세요.** 에러 메시지를 전체 복사해서 Claude한테 이렇게 물어보세요.

"다음과 같은 에러가 떴어. 정확한 이유를 알려주고, 어떻게 고치면 되는지 알려줘. [에러 메시지 붙여넣기]"

### 자주 나오는 에러

| 에러 메시지 | 의미 | 해결 방법 |
|-----------|------|---------|
| "Cannot find module" | 필요한 파일을 못 찾음 | `npm install` 입력 |
| "Port 3000 is already in use" | 포트가 이미 사용 중 | 다른 터미널 창 닫거나, 다른 포트 사용 |
| "Permission denied" | 권한 없음 | Mac: 명령어 앞에 `sudo` 붙이기 / Windows: 관리자 권한 실행 |
| "command not found" | 프로그램 못 찾음 | 터미널 닫고 다시 열기, 안 되면 재시작 |

### 세션이 날아갔을 때

세션(Claude와의 대화 기록)이 날아가도 **프로젝트 파일은 폴더에 그대로 남아있습니다.** 터미널을 다시 열고, 프로젝트 폴더로 이동한 뒤, `claude`를 입력하면 이어서 작업할 수 있습니다.

**미리 대비하는 법**

- 중요한 프로젝트는 폴더를 복사해두세요. "my-project-v1", "my-project-v2" 이런 식으로 버전을 남기면 언제든 이전 상태로 돌아갈 수 있습니다.
- Claude한테 뭘 요청했는지 메모장에 간단히 적어두면, 세션이 날아가도 빠르게 복구할 수 있습니다.
