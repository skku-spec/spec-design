# 고급 기능
<!-- new -->

> 코딩 에이전트를 더 효율적으로 활용할 수 있는 고급 기능을 소개합니다. Claude Code뿐 아니라 다른 코딩 에이전트에서도 비슷한 개념이 활용됩니다.
>
> 1. 스킬(Skill)
> 2. 서브에이전트(Sub-agent)
> 3. 랄프루프(Ralph Loop)

## 스킬(Skill)

스킬은 **코딩 에이전트에게 특정 분야의 전문 지식을 미리 주입하는 기능**입니다.

사람도 "프론트엔드 개발자"와 "백엔드 개발자"가 전문 분야가 다르듯, AI에게도 특정 작업에 대한 가이드라인을 미리 알려줄 수 있습니다. 이 가이드라인을 "스킬"이라고 합니다.

### 스킬이 필요한 이유

AI에게 매번 "이 프로젝트는 React를 쓰고, 이런 규칙을 따르고, 이런 폴더 구조를 갖고 있어..."라고 설명하면 비효율적입니다. 스킬을 한 번 설정해두면 AI가 **알아서 해당 규칙을 따릅니다**.

### 예시

- **프론트엔드 스킬** — React, Tailwind CSS, 컴포넌트 구조 등 프론트엔드 작업에 필요한 규칙을 미리 설정
- **디자인 스킬** — 디자인 시스템, 색상 팔레트, 타이포그래피 규칙을 미리 설정
- **Git 스킬** — 커밋 메시지 규칙, 브랜치 전략 등을 미리 설정

### 바이브 코딩에서의 활용

우리 같은 비개발자도 스킬을 활용하면, AI에게 "나는 이런 프로젝트를 하고 있어"라는 맥락을 한 번에 전달할 수 있습니다. 처음 프로젝트를 세팅할 때 스킬을 잘 설정해두면, 이후 작업이 훨씬 매끄러워집니다.

## 서브에이전트(Sub-agent)

서브에이전트는 **메인 AI가 다른 AI를 부하로 불러서 작업을 분배하는 기능**입니다.

### 왜 필요한가?

혼자서 모든 일을 하면 시간이 오래 걸립니다. 서브에이전트를 사용하면 메인 AI가 "팀장" 역할을 하면서, 세부 작업을 여러 "팀원" AI에게 동시에 맡길 수 있습니다.

### 동작 방식

1. 사용자가 메인 AI에게 큰 작업을 요청합니다.
2. 메인 AI가 작업을 분석하고 여러 개의 세부 작업으로 나눕니다.
3. 각 세부 작업을 서브에이전트에게 **동시에** 맡깁니다.
4. 서브에이전트들이 각자 작업을 완료하면 메인 AI가 결과를 종합합니다.

### 예시

"이 웹사이트에 로그인 페이지, 대시보드, 설정 페이지를 만들어줘"라고 요청하면:

- **서브에이전트 A** → 로그인 페이지 작업
- **서브에이전트 B** → 대시보드 작업
- **서브에이전트 C** → 설정 페이지 작업

이렇게 3개를 동시에 만들어서 훨씬 빠르게 완성할 수 있습니다.

## 랄프루프(Ralph Loop)

랄프루프는 **AI가 스스로 작업을 반복하면서 완성도를 높이는 자동 루프 기능**입니다.

### 일반 작업 vs 랄프루프

| 항목 | 일반 작업 | 랄프루프 |
|------|----------|---------|
| 작업 방식 | 사용자가 매번 "다음 해줘" 지시 | AI가 알아서 다음 단계 진행 |
| 오류 처리 | 사용자가 확인 후 수정 요청 | AI가 스스로 테스트하고 수정 |
| 완성도 | 사용자가 체크해야 함 | AI가 스스로 검증 반복 |
| 속도 | 사용자 응답 대기 시간 포함 | 연속 작업으로 빠름 |

### 동작 방식

1. 사용자가 목표를 지시합니다.
2. AI가 작업을 수행합니다.
3. AI가 스스로 결과를 검토합니다.
4. 부족한 점이 있으면 **스스로 수정합니다**.
5. 2~4를 반복하여 목표에 도달할 때까지 계속합니다.

### 주의사항

- 랄프루프를 켜면 AI가 오래 작업할 수 있으므로, **API 비용**을 주의해야 합니다.
- 처음에는 작은 작업부터 시도하고, 익숙해지면 범위를 넓히는 것이 좋습니다.
