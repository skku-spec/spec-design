## 컨텍스트 윈도우란?

컨텍스트 윈도우는 **AI가 한 번에 읽고 기억할 수 있는 정보의 양**입니다.

사람으로 비유하면, 시험 볼 때 **오픈북 시험에서 넘길 수 있는 페이지 수**와 같습니다. 페이지를 10장만 볼 수 있는 시험과 300장을 볼 수 있는 시험은 난이도가 완전히 다릅니다.

### 토큰이란?

AI는 글자를 "토큰"이라는 단위로 읽습니다. 대략적으로:

- 영어 1단어 ≈ 1토큰
- 한글 1글자 ≈ 1~2토큰
- **1,000토큰 ≈ A4 용지 1장** 정도입니다

### 컨텍스트가 왜 중요한가?

바이브 코딩에서 AI에게 프로젝트 전체를 이해시키려면, 프로젝트의 모든 파일을 컨텍스트 안에 넣어야 합니다.

| 컨텍스트 크기 | 할 수 있는 것 |
|--------------|-------------|
| 4,000토큰 | 짧은 대화, 간단한 질문 |
| 32,000토큰 | 긴 문서 1개 분석 |
| 128,000토큰 | 책 1권 분량 이해 |
| 200,000토큰 | 중소규모 프로젝트 전체 이해 |
| 1,000,000토큰 | 대규모 프로젝트, 여러 파일을 동시에 분석 |

컨텍스트가 크면 클수록 AI가 더 많은 코드를 한꺼번에 이해하고, 더 정확한 작업을 할 수 있습니다.

## 주요 모델 비교

현재 시점(2026년 3월) 기준, 주요 AI 모델 비교입니다.

### LLM (대화형 AI)

| 모델 | 회사 | 컨텍스트 | 특징 |
|------|------|---------|------|
| ChatGPT (GPT-5.2) | OpenAI | 400K | 가장 대중적. 다목적 |
| Claude (4.6) | Anthropic | 1M | 긴 글 분석, 코딩에 강점 |
| Gemini (3 Pro) | Google | 1M | 이미지 생성, 검색 연동 |

### 코딩 에이전트

| 모델 | 회사 | 방식 | 특징 |
|------|------|------|------|
| Claude Code | Anthropic | 로컬 터미널 | 대화하면서 실시간 코딩. 컨텍스트 1M |
| Codex | OpenAI | 클라우드 | 작업을 맡기고 기다리는 방식. 비용 효율적 |

Claude Code는 **2025년 2월** 리서치 프리뷰로 시작해, **2025년 5월** Claude 4와 함께 정식 출시되었습니다. Codex(코딩 에이전트 버전)는 **2025년 5월 16일**에 출시되었습니다.

**우리 팀은 Claude Code를 사용합니다.** 실시간으로 대화하면서 코딩할 수 있어 바이브 코딩에 가장 적합합니다.

## Claude 요금제 비교

![Claude 요금제 페이지](vibe/claude-pricing.png)

Claude는 3가지 요금제를 제공합니다. 요금제에 따라 사용할 수 있는 토큰 양이 달라집니다.

| 요금제 | 월 가격 | 사용 가능한 모델 | 특징 |
|--------|--------|----------------|------|
| **Free** | 무료 | Sonnet 4.6 | 하루 제한 있음. 가벼운 대화용 |
| **Pro** | $20 (약 2.7만원) | Opus 4.6 + Sonnet 4.6 | 일반 사용자 기준 충분한 사용량 |
| **Max 5x** | $100 (약 13.5만원) | Opus 4.6 + Sonnet 4.6 | Pro 대비 5배 사용량 |
| **Max 20x** | $200 (약 27만원) | Opus 4.6 + Sonnet 4.6 | Pro 대비 20배 사용량 |

### 무엇이 다른가?

핵심 차이는 **한 달에 쓸 수 있는 토큰 총량**입니다.

- **Pro($20)**: 일반적인 대화와 코딩 작업에 충분합니다. 하루에 2~3시간 정도 Claude Code를 사용한다면 이 플랜으로 충분합니다.
- **Max 5x($100)**: 하루 종일 Claude Code를 돌리거나, 큰 프로젝트를 빠르게 진행할 때 필요합니다.
- **Max 20x($200)**: 전문 개발자가 Claude Code를 메인 도구로 사용하는 경우입니다. 랄프루프처럼 AI가 자동으로 반복 작업하면 토큰 소모가 크기 때문에, 이런 기능을 많이 쓰면 필요할 수 있습니다.

## 토큰 소모 실전 예시

"토큰이 얼마나 드는지" 감이 잘 안 올 수 있습니다. 실제 작업별 토큰 소모량을 정리했습니다.

### 일반 대화

| 작업 | 대략적인 토큰 소모 | 비유 |
|------|------------------|------|
| "안녕, 오늘 뭐 할까?" 짧은 대화 1회 | 500~1,000토큰 | A4 반 장 |
| 긴 질문 + 긴 답변 1회 | 3,000~5,000토큰 | A4 3~5장 |
| 30분 동안 이어진 대화 | 20,000~50,000토큰 | A4 20~50장 |
| 에세이 첨삭 요청 (A4 3장 분량) | 10,000~15,000토큰 | A4 10~15장 |

### 바이브 코딩 (Claude Code)

| 작업 | 대략적인 토큰 소모 | 비유 |
|------|------------------|------|
| "버튼 색상 바꿔줘" 같은 간단한 수정 | 5,000~10,000토큰 | A4 5~10장 |
| "로그인 페이지 만들어줘" 중간 작업 | 30,000~80,000토큰 | A4 30~80장 |
| "이 웹사이트 전체 구조 잡아줘" 큰 작업 | 100,000~300,000토큰 | A4 100~300장 |
| 랄프루프로 자동 반복 작업 (1시간) | 200,000~500,000토큰 | A4 200~500장 |
| **이 교육자료 사이트 만드는 작업 (오늘 세션)** | **1,000,000토큰 이상** | **A4 1,000장 이상** |

### 왜 바이브 코딩은 토큰을 많이 쓸까?

일반 대화는 질문과 답변만 왔다갔다 합니다. 하지만 바이브 코딩에서 Claude Code는:

1. **프로젝트 파일 전체를 읽습니다** — 폴더 안의 모든 코드를 이해해야 합니다.
2. **코드를 생성합니다** — 코드 자체가 긴 텍스트입니다.
3. **결과를 확인합니다** — 실행 결과, 에러 메시지 등을 분석합니다.
4. **반복합니다** — 수정 → 확인 → 재수정 루프가 계속됩니다.

**결론**: Pro($20) 플랜이면 가벼운 바이브 코딩에 충분합니다. 본격적으로 프로젝트를 진행한다면 Max($100~200)를 고려하면 됩니다.
