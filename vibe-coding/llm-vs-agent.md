## LLM이란?

LLM은 **Large Language Model**, 대규모 언어 모델입니다. 쉽게 말하면 **대화형 AI**입니다. 질문하면 답해주고, 글을 써주고, 코드를 알려주는 AI입니다.

### 대표적인 LLM

1. **ChatGPT** (OpenAI) — 가장 대중적인 AI. 범용적으로 활용 가능합니다.
2. **Claude** (Anthropic) — 긴 글 이해, 분석, 코딩에 강점이 있습니다.
3. **Gemini** (Google) — 이미지 생성(나노바나나), 검색 연동이 강점입니다.

### LLM의 한계

LLM은 **대화만** 할 수 있습니다. 코드를 알려줄 수는 있지만, 직접 파일을 만들거나 프로그램을 실행하지는 못합니다. 마치 "이렇게 하면 돼"라고 말만 해주는 선배와 같습니다.

## 코딩 에이전트란?

코딩 에이전트는 LLM에 **실행 능력**을 더한 것입니다. 대화만 하는 게 아니라, 직접 파일을 만들고, 코드를 수정하고, 프로그램을 실행할 수 있습니다.

### 대표적인 코딩 에이전트

1. **Claude Code** (Anthropic) — 터미널에서 동작하는 코딩 에이전트. 파일 생성/수정, 명령어 실행, 프로젝트 전체 이해가 가능합니다. 우리 팀이 사용하는 도구입니다.
2. **Codex** (OpenAI) — ChatGPT 기반의 코딩 에이전트. 클라우드에서 코드를 실행합니다.

### LLM vs 코딩 에이전트

| 항목 | LLM | 코딩 에이전트 |
|------|-----|--------------|
| 대화 | ✅ | ✅ |
| 코드 작성 | 알려주기만 함 | 직접 파일에 작성 |
| 파일 생성/수정 | ❌ | ✅ |
| 프로그램 실행 | ❌ | ✅ |
| 프로젝트 이해 | ❌ (복붙해야 함) | ✅ (폴더 전체를 읽음) |
| 비유 | 말만 하는 선배 | 직접 해주는 선배 |

**핵심 차이**: LLM에게 "로그인 페이지 만들어줘"라고 하면 코드를 **채팅으로 보여줍니다**. 코딩 에이전트에게 같은 말을 하면 실제 파일을 **만들어서 저장합니다**.

바이브 코딩은 이 코딩 에이전트를 활용하는 것입니다. 그래서 우리는 Claude Code를 사용합니다.
